# Bubble Chat — P2P 실시간 채팅 앱

접속 코드로 누구와나 즉시 연결하는 PeerJS 기반 P2P 실시간 채팅 애플리케이션입니다.

## 📋 개요

**특징:**
- ✅ 호스트-게스트 P2P 아키텍처 (Star Topology)
- ✅ QR Code 접속 지원
- ✅ 메시지 히스토리 자동 저장 (localStorage)
- ✅ DM(Direct Message) 기능
- ✅ 모바일 반응형 UI
- ✅ 다크모드 지원
- ✅ 채팅 내용 다운로드 (.txt)

---

## 🏗️ 아키텍처

### PeerJS 기반 P2P 통신
```
Host (Peer ID: bubble-{roomCode})
  ├─ Guest 1 (bubble-guest-xxx)
  ├─ Guest 2 (bubble-guest-yyy)
  └─ Guest N (bubble-guest-zzz)
```

**룸 코드**: 6자리 숫자 (100000–999999)
- 호스트가 생성
- 게스트가 입력하여 호스트에 연결

### 세션 저장소

**localStorage 키:**
- `bubbleChat_activeSessions` - 진행 중/일시중단 대화
- `bubbleChat_completedSessions` - 완전 종료 대화
- `bubbleChat_guestNicknames` - 게스트 닉네임 캐시

**세션 구조:**
```javascript
{
  code: "123456",                      // 룸 코드
  roomName: "수학 토론방",
  role: "host" | "guest",
  status: "active" | "paused" | "completed",
  participants: ["Alice", "Bob"],      // 닉네임 배열
  hostNickname: "Alice",               // DM 대상 식별용
  messages: [...],                     // 메시지 히스토리
  messageCount: 42,
  startedAt: "2026-04-04T10:30:00.000Z",
  pausedAt: "2026-04-04T10:45:00.000Z" // 일시중단 시에만
}
```

## 🔄 사용 흐름

### 호스트 흐름

```
1. 홈 화면
   ↓
2. "[+ 새 대화]" 클릭 → 모달 열기
   ├─ 방 이름 입력: "수학 토론"
   ├─ 닉네임 입력: "선생님"
   └─ [시작] 클릭
   ↓
3. 코드 자동 생성 → 모달에 표시
   ├─ 코드: 654321
   ├─ QR Code 생성 (스캔 가능)
   └─ 게스트 대기 중...
   ↓
4. 홈 화면에 카드 추가 (상태: "🟢 진행 중")
   ↓
5. 게스트 입장 → 참여자 목록 업데이트
   ├─ Alice (호스트)
   └─ Bob (게스트)
   ↓
6. 메시지 송수신
   ├─ 전체 메시지 (모두에게)
   └─ DM (특정 게스트)
   ↓
7. "[대화 종료]" 클릭 → 선택 모달
   ├─ "[일시 중단]" → 카드 유지 (⏸️ 일시 중단)
   │  └─ 게스트도 카드 유지, 재진입 가능
   │
   └─ "[완전 종료]" → 카드는 호스트만 보임 (🔒 완전 종료)
      └─ 게스트는 카드 자동 삭제
      └─ 게스트 메시지 받지 않음
```

### 게스트 흐름

```
1. 홈 화면
   ↓
2. "[+ 새 대화]" 클릭 → 모달 열기
   ├─ 방법 1: 코드 입력 (654321)
   ├─ 방법 2: QR Code 스캔
   │  └─ URL ?code=654321 → 자동 입력
   ├─ 닉네임 입력: "Bob"
   └─ [입장] 클릭
   ↓
3. 호스트 연결 → 핸드셰이크
   ├─ 코드 확인: 유효한가?
   ├─ 닉네임 중복 확인: 호스트와 같은가?
   └─ 참여자 목록 수신
   ↓
4. "연결되었습니다" 토스트 표시
   ↓
5. 호스트의 "시작 신호" 대기
   ├─ 카드 상태: "🟢 진행 중"
   └─ 대화창 자동 오픈
   ↓
6. 메시지 송수신
   ├─ 전체 메시지 → 모두에게
   └─ DM → 호스트에게만 (➡️ Host)
   ↓
7. "[대화에서 나가기]" 클릭
   ├─ 연결 종료
   ├─ 상태 저장 ("🟢 진행 중" 유지)
   └─ 홈으로 복귀
   ↓
8. 호스트의 "완전 종료" 신호 받음
   ├─ 토스트: "호스트가 대화를 종료했습니다"
   ├─ 카드 자동 삭제
   └─ 홈으로 복귀
```

## 💬 메시지 타입

### Chat (전체 메시지)
```javascript
{
  type: "chat",
  from: "Bob",
  to: null,
  content: "안녕하세요!",
  timestamp: 1717500000000
}
```
→ 모든 참여자가 받음

### DM (Direct Message)
```javascript
{
  type: "dm",
  from: "Bob",
  to: "Alice",              // 호스트 이름
  content: "선생님, 질문있어요",
  timestamp: 1717500000000
}
```
→ 렌더링: `➡️ Alice: [메시지]`
→ 호스트와 Bob만 받음

### System (시스템 메시지)
```javascript
{
  type: "system",
  from: "system",
  to: null,
  content: "Bob님이 입장했습니다",
  timestamp: 1717500000000
}
```
→ 가운데 정렬, 회색 이탤릭 텍스트

## ⚙️ 핵심 기능

### 1. 세션 관리
- **활성 세션** (activeSessions): 진행 중, 일시중단 대화
- **완료 세션** (completedSessions): 호스트만 열람 가능 (삭제됨)
- **상태 값**:
  - `active`: 진행 중
  - `paused`: 호스트가 일시중단
  - `completed`: 호스트가 완전 종료

### 2. DM 드롭다운
- **호스트**: 게스트 목록 표시 (활성 게스트만)
- **게스트**: 호스트만 표시 (➡️ Host)

### 3. 참여자 관리
```javascript
{
  nickname: "Bob",
  isHost: false,
  joinTime: 1717500000000,      // 입장 시간
  leaveTime: 1717500060000      // 퇴장 시간 (null이면 활성)
}
```
→ **UI 표시**: "입장: 10:30 / 퇴장: 10:31"
→ **시각 구분**: 퇴장한 참여자는 60% 투명도

### 4. QR Code 접속
1. 호스트: 코드 클릭 → QR Code 모달 표시
2. 게스트: QR Code 스캔 → `?code=123456` URL
3. 자동 입력: 코드 필드 자동 채워짐
4. 닉네임만 입력하면 바로 입장

### 5. 채팅 내용 저장 (.txt)
- **호스트 전용** 기능
- 메타데이터 포함:
  - 방 이름, 코드, 역할
  - 시작 시간
  - 참여자 명단
  - 참여/퇴장 시간
- 메시지 형식:
  ```
  [10:30] Alice: 안녕하세요
  [10:31] Bob ➡️ Alice: 질문있어요
  [10:32] [시스템] Bob님이 입장했습니다
  ```

## 📝 개발 노트

### 상태 관리
```javascript
STATE = {
  role: 'host' | 'guest',
  roomCode: '123456',
  roomName: '수학 토론',
  nickname: 'Bob',
  peer: Peer,                     // PeerJS instance
  hostConn: DataConnection,       // 게스트→호스트
  guestConns: { [name]: conn },   // 호스트→게스트들
  participants: [...],
  messages: [...],
  sessionStart: timestamp,
  chatStarted: boolean,
  chatEnded: boolean,             // 호스트: 메시지 무시
  isClosing: boolean,             // 중복 저장 방지
}
```

### 메시지 흐름
1. **호스트 → 게스트 (1:N)**
   - `broadcastMessage()` 호출
   - 모든 게스트 연결에 전송

2. **게스트 → 호스트**
   - `hostConn.send()` 호출
   - 호스트가 relay (다른 게스트에게 전송)

3. **중복 방지**
   - timestamp + content로 중복 체크
   - 자신의 메시지가 relay되어 돌아올 때 무시
