## 🧪 향후 검증 계획

### 1. E2E 테스트 자동화

**목표**: 사용자 시나리오를 자동으로 검증하여 회귀 테스트 방지

**도구**: Playwright 또는 Cypress
- 다중 브라우저 지원 (Chrome, Firefox, Safari, Edge)
- 헤드리스 모드 실행 가능
- 스크린샷 및 비디오 녹화

**테스트 시나리오**:
```
1. 호스트 방 생성 → 게스트 입장 → 메시지 송수신 → 종료
   - 호스트: 방 이름 입력 → 코드 생성 → 게스트 대기
   - 게스트: 코드 입력 → 입장 → 메시지 전송
   - 호스트: 메시지 수신 확인 → 일시중단 선택

2. 게스트 재입장 시나리오
   - 게스트: 페이지 새로고침 → 같은 코드로 재입장
   - 호스트: 게스트 카드 재표시 확인

3. 다중 게스트 동시 접속
   - 호스트: 3명의 게스트 입장 대기
   - 게스트들: 동시에 메시지 전송
   - 호스트: 모든 메시지 수신 및 relay 확인

4. 완전 종료 시나리오
   - 호스트: "완전 종료" 선택
   - 게스트: 카드 자동 삭제 확인
   - 게스트: 같은 코드 재입장 시도 → 차단 확인
```

### 2. 성능 테스트

**목표**: 메시지 대량 송수신, 장시간 연결 시 안정성 검증

**테스트 항목**:
- **메시지 대량 송수신**: 1000개 이상 메시지 처리
  - 렌더링 성능 (프레임 드롭 확인)
  - 메모리 사용량 (Chrome DevTools)
  - localStorage 용량 (5-10MB 제한)

- **장시간 연결**: 1시간 이상 유지
  - 메모리 누수 확인
  - 네트워크 안정성
  - 브라우저 탭 포커스 전환

- **메모리 누수 체크**:
  - Chrome DevTools Memory Profiler
  - 반복적인 메시지 송수신 후 가비지 컬렉션 확인

### 3. 호환성 테스트

**목표**: 다양한 환경에서 동작 확인

**브라우저 호환성**:
- Chrome (Latest, Latest-1)
- Firefox (Latest, Latest-1)
- Safari (Latest, Latest-1)
- Edge (Latest, Latest-1)

**모바일 호환성**:
- iOS Safari (Latest)
- Android Chrome (Latest)
- 화면 크기: 320px ~ 1920px (여러 breakpoint)

**네트워크 조건**:
- DevTools Network Throttle
  - Slow 3G (400 kbps)
  - 4G/LTE (4 Mbps)
  - WiFi (30 Mbps)
- 네트워크 끊김 → 복구 시뮬레이션

### 4. 보안 검토

**목표**: XSS, CORS, 데이터 누수 등 보안 취약점 확인

**검토 항목**:
- **XSS 취약점**:
  - 메시지 입력 (HTML/JavaScript 코드 포함)
  - 닉네임 입력
  - 방 이름 입력
  - 렌더링 시 적절한 이스케이프 확인

- **CORS 설정**:
  - PeerJS CDN 로드 가능 확인
  - 외부 리소스 로드 보안

- **데이터 보호**:
  - localStorage 민감 정보 저장 여부 확인
  - 세션 정보 암호화 (선택)

### 5. 다중 세션 검증 전략

**사용자 피드백**: "두 세션 이상에서 점검하려니 쉽지 않음"
→ 검증 방법 다양화

**로컬 검증**:
- Chrome + Firefox 동시 실행
- 동일 머신, 다른 브라우저
- DevTools Network Throttle으로 느린 네트워크 시뮬레이션

**원격 검증** (선택):
- 휴대폰 + 데스크톱 동시 접속
- 다른 네트워크 (WiFi + 모바일 데이터)
- Ngrok 또는 로컬 터널링으로 외부 기기 접속 지원

**자동화 검증**:
- Playwright 멀티 탭 시나리오
- 각 탭별 콘솔 로그 수집
- 스크린샷 동시 캡처

## 🔧 기술적 고려사항

### 현재 구조 제약

**localStorage 5-10MB 제한**
- 현재: 최대 20개 세션 저장
- 해결 방안:
  1. 장기 해결: IndexedDB로 확장 저장소 구현
  2. 단기 해결: 오래된 세션 자동 삭제 (30일 이상)
  3. 메시지 압축 또는 서버 저장

**PeerJS 신뢰성**
- STUN/TURN 서버 선택에 따라 연결성 달라짐
- NAT/방화벽 환경에서 일부 네트워크 접속 불가
- 해결 방안:
  1. 다중 STUN/TURN 서버 설정
  2. 폴백 메커니즘 구현 (P2P 실패 시 서버 relay)

**메시지 동기화**
- 게스트 재입장 시 이전 메시지 미동기
- 호스트만 메시지 히스토리 보유
- 해결 방안:
  1. 중앙 메시지 서버 도입 (Node.js)
  2. 게스트 입장 시 서버에서 히스토리 다운로드

### 성능 최적화 기회

**1. 메시지 가상 스크롤링**
- 문제: 1000+ 메시지 렌더링 시 느려짐
- 해결: 가상 스크롤(Virtual Scroll) 라이브러리 도입
- 예상 효과: 90% 성능 개선

**2. 이미지 지연 로딩**
- 문제: 파일 공유 시 대량 이미지 렌더링
- 해결: Intersection Observer API로 화면 내 이미지만 로드
- 예상 효과: 초기 로딩 시간 50% 단축

**3. 번들 크기 최적화**
- 현재: index.html ~2800 lines (~150KB)
- 목표: 코드 분할(Code Splitting), 자동화 빌드 도입
- 예상 효과: 20-30% 크기 감소

### UX 개선 기회

**1. 타이핑 표시**
- 구현: `type: 'typing'` 메시지로 "XXX님이 입력 중..." 표시
- 예상 복잡도: Low
- 예상 시간: 1주

**2. 메시지 시간 자동 갱신**
- 구현: setInterval()로 시간 재계산 (상대 시간 유지)
- 예상 복잡도: Low
- 예상 시간: 몇 시간

**3. 스크롤 자동 정지 해제**
- 현재: 스크롤 시 자동 하단 정지 해제 안 됨
- 구현: 스크롤 감지 후 메시지 도착 시에만 하단 정지
- 예상 복잡도: Low-Medium
- 예상 시간: 1-2일
